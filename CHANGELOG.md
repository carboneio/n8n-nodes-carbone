> 19 March 2026

##### ⚠️ Breaking change — Generate Document: Template Source

The `Use Inline Template File (Base64)` toggle (boolean) has been replaced by a `Template Source` selector (options field) with three choices:

- **Template ID** *(default)* — Use a template stored on Carbone. Select from the list or enter an ID. Same as `templateSource: false` in v1.x.
- **File** — Use a template file from a previous n8n node (DOCX, XLSX, PPTX, ODT, HTML…). The file is encoded to base64 automatically — no manual conversion required.
- **Base64 String** — Provide the template as a base64-encoded string directly. Same as `templateSource: true` in v1.x.

**Why this improves UX and usability:**

Previously, users who wanted to generate a document from a local or dynamically-fetched template file had to manually convert it to base64 first — typically using a Code node or a dedicated conversion step. This created friction and was error-prone (wrong encoding, missing padding, data: URI prefix issues).

With `Template Source: File`, users can now connect any node that outputs a binary file — such as an HTTP Request, Read/Write Binary Files, Google Drive, Dropbox, or a webhook — directly to the Carbone node. The base64 encoding happens internally. This eliminates the extra conversion step and makes the integration accessible to No/Low-code users who are not familiar with base64.

**Migration guide:**
- `templateSource: false` (stored template) → select **Template ID**
- `templateSource: true` (base64 string) → select **Base64 String** (identical behavior)
- New workflows → use **File** to skip manual base64 encoding
